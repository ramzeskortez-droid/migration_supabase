ВЕРСИЯ 1.13.0 - Интеграция с Bitrix24 и рефакторинг воронки

- Добавлена Edge Function `create-bitrix-deal` для автоматического создания сделок в Bitrix24 при отправке КП клиенту.
- Реализована логика сопоставления контактов по номеру телефона и передачи состава заказа в CRM.
- Переработана структура вкладок у Менеджера: "Готов купить" заменен на "КП у клиента".
- Статус "КП готово" теперь разделен на два этапа: сформированное (внутреннее) и отправленное (внешнее).
- Исправлены баги с исчезновением заказов из списков у Менеджера и Оператора.
- Обновлены счетчики заказов в `SupabaseService` для корректного отображения суммы статусов.

БЫЛА ПРОБЛЕМА:
- Отсутствовала автоматизация передачи данных в CRM Bitrix24.
- Заказы пропадали из интерфейса Менеджера сразу после того, как Оператор нажимал кнопку "Отправлено клиенту".
- Возникали ошибки при работе Edge Functions с тестовыми доменами из-за просроченных SSL-сертификатов.

КАК РЕШИЛИ:
- Создан асинхронный триггер (Webhook), запускающий облачную функцию при смене статуса на "КП отправлено".
- В функции реализован переход на HTTP для обхода SSL-блокировок и добавлена обработка NULL-значений `user_id`.
- Расширены фильтры `getStatusFilter` в интерфейсах для поддержки многостадийных табов.
- Добавлен принудительный `refetch` для исключения проблем с кешированием данных.
