ВЕРСИЯ 1.18.39 - Рефакторинг Сервисов и Улучшение Файлов

1. Рефакторинг SupabaseService:
   - Начата декомпозиция монолита. Выделены модули: `api/finance`, `api/storage`, `api/brands`, `api/settings`.
   - Основной класс теперь делегирует выполнение методов.

2. Управление файлами (Оператор и Менеджер):
   - Реализована мульти-загрузка файлов для позиций заказа во всех интерфейсах.
   - Внедрен единый компонент `FileDropzone` с поддержкой Drag&Drop.
   - Файлы позиций теперь агрегируются в общем блоке "Файлы по заявке" у оператора.
   - Добавлена возможность удаления файлов позиций из общего списка.
   - Улучшен UI: кнопки удаления появляются только при наведении.

3. Финансы и UI:
   - Исправлен баг с перезаписью стейта при загрузке нескольких файлов подряд.
   - Обновлена шапка сайта: курсы валют теперь отображаются с запятой, убран лишний курс из БД, оставлены только настроечные пары.

БЫЛА ПРОБЛЕМА:
1. `SupabaseService.ts` стал слишком большим и сложным для поддержки.
2. Оператор не мог загрузить несколько файлов к одной позиции (они перезаписывались).
3. Менеджер не мог редактировать файлы позиций.
4. Курсы валют в шапке отображались некорректно (старые данные из БД).

КАК РЕШИЛИ:
1. Разнесли логику по папкам `src/services/api/*`.
2. Переписали `PartsList` и `OrderFilesUpload` для корректной работы с массивами файлов.
3. Добавили `FileDropzone` в таблицу менеджера.
4. Переписали логику рендера курсов в `App.tsx`.
