ВЕРСИЯ 1.18.17 - Расширение функционала Закупщика: Табы и Сортировка
- Реализована динамическая сортировка в листинге закупщика по колонкам "№ заказа", "Срок до", "Статус" и "Дата".
- Установлена сортировка по умолчанию по полю "Срок до" (deadline) в порядке возрастания.
- В интерфейс закупщика добавлены новые табы для улучшения навигации:
  1. "В торгах" (активные заявки с поданными офферами).
  2. "Выигранные" (заявки, где закупщик победил).
  3. "Проигранные" (закрытые заявки без победы).
  4. "Отмененные" (аннулированные заказы).
- Обновлена логика SupabaseService.getOrders и getBuyerTabCounts для поддержки новых фильтров.
- Исправлен баг, при котором параметры сортировки игнорировались API.
- Устранены ошибки дублирования идентификаторов и импортов в коде компонентов.
- Исправлена верстка панели вкладок: скрыты системные скроллбары (добавлен класс no-scrollbar), улучшено позиционирование кнопки обновления.

БЫЛА ПРОБЛЕМА:
Закупщику было сложно ориентироваться в большом потоке заявок: "Отправленные" смешивались в кучу, сортировка не работала, а статус заказа был неочевиден. Визуальный шум (скроллбары) портил интерфейс.

КАК РЕШИЛИ:
1. Разделили единый список "Отправленные" на смысловые категории (Табы).
2. Внедрили маппинг сортировки UI -> DB в сервисном слое.
3. Обновили UI тулбара и таблицы для поддержки новых функций.
4. Добавили CSS-утилиты для скрытия скроллбаров.
