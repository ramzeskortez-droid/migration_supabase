# Version History

## 1.18.75 - Обновление Оператора (Почта, Торги, Ручная обработка)
- **Email Workflow:**
  - Реализован механизм **блокировки писем** (Locking) для предотвращения коллизий между операторами. При взятии в работу письмо блокируется на 60 секунд.
  - Внедрена **привязка письма к заказу**: письмо остается в "Новых" до момента создания заявки, затем автоматически уходит в Архив.
  - В архиве почты теперь отображается яркая плашка с номером созданного заказа.
  - Панель почты теперь открыта по умолчанию для быстрого старта.
- **Operator UI:**
  - Добавлен таб **"Идут торги"**: отдельный список для заказов, по которым получены офферы (синий статус).
  - Внедрен статус **"Обработано вручную"** (фиолетовый) для завершения сделок в ручном режиме.
  - Удален селектор ролей из шапки (упрощение UI).
- **Database:**
  - Добавлена связь `orders.email_message_id` -> `incoming_emails.id`.
  - Добавлены поля блокировки `locked_by`, `locked_at` в `incoming_emails`.

## 1.18.74 - Рефакторинг интерфейса Оператора (Табы и Статусы)
- **Operator UI:**
  - Удалены табы "Успешные", "Отказ" и логика статуса "ГОРИТ" для оператора.
  - Таб "Обработано" переименован в "КП готово" (фильтр строго по этому статусу).
  - Реализован переход выполненных заказов ("Отправлено клиенту") в Архив.
  - Добавлено сворачивание списка файлов в шапке заказа (по умолчанию скрыты).
  - Обновлена цветовая схема статусов: "КП готово" (желтый), "КП отправлено" (зеленый).
  - Добавлено правило Cross-Impact Analysis в rules.md.
