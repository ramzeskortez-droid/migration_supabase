ВЕРСИЯ 1.13.1 - Багфиксы и Optimistic UI

- Исправлен критический баг "связанных" лидеров: теперь выбор победителя работает по уникальным ID, а не по названиям товаров.
- Решена проблема с перемешиванием строк после обновления благодаря сортировке по ID в getOrderDetails.
- Реализовано "Оптимистичное обновление" (Optimistic UI): мгновенный визуальный отклик при выборе лидера.
- Разрешен выбор нескольких победителей для одной позиции заказа.
- Оптимизирована логика сопоставления офферов с товарами в AdminItemsTable.

БЫЛА ПРОБЛЕМА:
- При наличии товаров с одинаковыми именами выбор лидера для одного влиял на все остальные.
- После выбора лидера товары в таблице менялись местами, вызывая визуальную путаницу с ценами.
- Задержка в 1-2 секунды при выборе лидера раздражала пользователей.

КАК РЕШИЛИ:
- Перевели updateRank на использование offer_item_id и order_item_id.
- Добавили .sort((a, b) => a.id - b.id) в сервисный слой получения деталей заказа.
- Использовали queryClient.setQueryData для мгновенного обновления локального кеша React Query.
